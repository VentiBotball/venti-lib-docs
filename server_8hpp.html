◆ get_position()

crow::response get_position ( const crow::request & req)

Handles an HTTP GET request to retrieve the robot's current position.

Returns a JSON object containing the current navigation state of the robot, including the current node, planned path, and offset vector.

Example JSON response:

{
"current_node": 5,
"path": [5, 6, 7],
"offset_vector": { "x": 0.1, "y": -0.03 }
}
Parameters
reqThe incoming HTTP request (unused in this case).
Returns
crow::response JSON response containing the current robot position.

"to": 2
}

◆ request_to_position()

crow::response request_to_position ( const crow::request & req)

Handles an HTTP POST request to move the robot to a specified target position.

The request body must contain a JSON object with target node or coordinate data.

Example request body:

{
"target_node": 10,
"offset_vector": { "x": 0.1, "y": -0.03 }
}
Parameters
reqThe HTTP request containing the target position in JSON format.
Returns
crow::response JSON response indicating success or failure.

◆ set_custom()

crow::response set_custom ( const crow::request & req)

Handles an HTTP POST request to set custom configuration parameters for the robot.

Accepts a JSON object with configuration key-value pairs such as max speed or behavior flags.

Example request body:

{
"pom_colors": ["RED", "YELLOW", "RED", "ORANGE"]
}
Parameters
reqThe HTTP request with custom configuration values.
Returns
crow::response JSON response indicating whether the configuration update was successful.

Configures the Crow web application with all robot-related HTTP routes.

Registers the routes for position retrieval, movement, outage reporting, and configuration. This should be called during the web server's initialization.
